Layer 2 — Content Script (content.js) — Runs in Every Matched Tab
🎓 Greeting Banner
showGreeting()
Shown 1.5s after page load. Displays site name. Auto-dismisses in 6s. "OPEN →" triggers widget with prefilled query.
DOM Overlay
🔘 Floating Action Button
injectFAB()
Persistent 🎓 button fixed bottom-right. Toggles widget open/close. Changes to ✕ when widget is open.
DOM Button
🪟 Shadow DOM Widget Panel
injectShadowWidget()
380px fixed right-side panel. Isolated via Shadow DOM (mode: open). Contains textarea, response area, smart doc cards, CTA link, and settings link.
Shadow DOM
🖱 Text Selection Detector
handleSelectionEvent()
Listens mouseup + pointerup (capture phase). 500ms debounce. Shows "✦ ASK AI" floating bubble near cursor if ≥6 chars selected.
Event Listener
📝 Input Field Detector
focusin / extractFieldContext()
300ms debounce on focusin. Extracts label text from: label[for], parent label, sibling label, placeholder/name/id. Shows "WHAT TO ENTER HERE?" hint bubble.
Event Listener
📤 submitQuery()
Query Sender
Reads textarea. Builds payload: {url, selectedText, fieldContext, userQuery}. Sends chrome.runtime.sendMessage ASK_GEMINI. 30s safety timer. Renders response with formatResponse().
Chrome Messaging
chrome.runtime.sendMessage → onMessage
Layer 3 — Background Service Worker (background.js) — Message Router & API Orchestrator
📨 Message Router
onMessage Listener
Routes 3 message types:
• ASK_GEMINI → handleQuery()
• GET_SETTINGS → chrome.storage.local
• OPEN_OPTIONS → openOptionsPage()
Service Worker
⚙ Settings Store
chrome.storage.local
Persists user config:
• groqKey (API key)
• backendUrl (profile API)
• docsUrl (documents API)
• username
Local Storage
⚡ Session Cache
chrome.storage.session
Short-lived cache per browser session:
• userProfile (fetched once)
• userDocuments (fetched once)
Avoids redundant API calls
Session Cache
🔧 handleQuery()
Main Orchestrator
1. Read settings from storage
2. Check session cache
3. Fetch profile + docs if needed
4. buildPrompt()
5. Call Groq API (25s timeout)
6. matchDocuments()
7. Return {answer, relevantDocs, cta}
Core Logic
📝 buildPrompt()
Prompt Constructor
System prompt: ScholarAssist persona + student profile data + uploaded documents list + response format rules.

User message: fieldContext + selectedText + page URL + question.

Temp: 0.4 · Max tokens: 1024
Prompt Engineering
🔍 matchDocuments()
Smart Doc Matching
Matches user's own words against DOC_KEYWORDS map.

10 keyword groups: aadhaar, income_cert, marksheet_10, marksheet_12, caste_cert, domicile, photo, bank_passbook, bonafide, caste_validity
Keyword Matching
