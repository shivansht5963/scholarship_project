PROJECT OVERVIEW
================================================================================
Scholar Match is an AI-powered scholarship management and matching platform 
designed to connect students with scholarships through intelligent eligibility 
matching, moderator verification, and a gamified karma points system.

TECH STACK
- Framework: Django 5.2.6
- Database: SQLite3 (Development) / PostgreSQL (Production-ready)
- Language: Python 3.x
- Frontend: HTML5, CSS3, JavaScript, TailwindCSS
- Architecture: Monolithic MVC Pattern


SYSTEM ARCHITECTURE
================================================================================

1. CORE MODULES
---------------

A. Users Module (users/)
   - Custom authentication system with role-based access
   - User types: Students, Moderators, Organizations
   - One-Time Registration (OTR) workflow for students

B. Scholarships Module (scholarships/)
   - Master scholarship database
   - Eligibility criteria engine
   - Required documents checklist

C. Applications Module (applications/)
   - Application lifecycle management
   - Document upload and verification
   - Roadmap-based progress tracking

D. Karma System Module (karma/)
   - Points-based gamification system
   - Scholarship submission (finder bounty program)
   - Rewards marketplace with redemption system
   - Leaderboard and ranking

E. Moderator Panel Module (moderator_panel/)
   - Application review queue
   - Document verification interface
   - Activity logging and audit trail

F. Funder Portal Module (funder_portal/)
   - Organization dashboard
   - Scholarship management
   - Application review and decision-making

G. AI Integrations Module (ai_integrations/)
   - Eligibility matching engine
   - Document OCR and validation
   - Audit logging for AI operations


2. DATABASE SCHEMA
================================================================================

A. USER MANAGEMENT
------------------
User (Custom AbstractUser)
- Fields: username, email, password, is_student, is_moderator, is_organization
- Purpose: Base authentication with role flags

StudentProfile (1-to-1 with User)
- Personal: full_name, dob, gender, phone, address
- Family: father_name, mother_name, annual_income, caste_category
- Identity: aadhaar_number, bank details (account, IFSC, bank_name)
- OTR Tracking: otr_completed, otr_step (1-7), profile_completion %
- Karma: total_karma_points, karma_rank, verified_scholar_badge
- Purpose: Comprehensive student data for eligibility matching

AcademicRecord (Many-to-1 with StudentProfile)
- Fields: degree_level, stream, institution_name, current_year, last_exam_score
- Purpose: Educational history for scholarship matching

StudentDocument (Many-to-1 with StudentProfile)
- Fields: document_type, file, uploaded_at, is_verified, notes
- Types: Aadhaar, income cert, caste cert, marksheets, bank passbook, etc.
- Purpose: Document repository for OTR and applications

ModeratorProfile (1-to-1 with User)
- Fields: organization_name, contact_phone, is_verified
- Purpose: NGO representatives or teacher accounts

OrganizationProfile (1-to-1 with User)
- Fields: organization_name, contact_person, official_email, website_url, is_verified_funder
- Purpose: Scholarship funding organizations


B. SCHOLARSHIP MANAGEMENT
--------------------------
Scholarship
- Core: name, organization, source_url, deadline, award_amount, details
- Status: is_active, is_verified
- Finder Bounty: found_by_student (FK), finder_karma_awarded
- Purpose: Master scholarship database

EligibilityCriteria (Many-to-1 with Scholarship)
- Fields: criterion_type, comparison_operator (GT/LT/EQ/IN), value, is_required
- Examples: MIN_INCOME < 250000, MIN_GPA >= 3.5
- Purpose: AI matching rules engine

RequiredDocument (Many-to-1 with Scholarship)
- Fields: document_name, description, is_mandatory
- Purpose: Application document checklist


C. APPLICATION LIFECYCLE
-------------------------
Application (Student + Scholarship combination)
- Fields: student, scholarship, status, moderator_assigned, application_link
- Status Flow: DRAFT → PENDING → MOD_VERIFIED → SUBMITTED → APPROVED/REJECTED
- Constraint: unique_together (student, scholarship) - one application per scholarship
- Purpose: Track student applications

UploadedDocument (Many-to-1 with Application)
- Fields: application, document_type, file, ai_verification_status, moderator_verified, moderator_notes
- AI Status: PENDING → AI_OK / AI_FLAG → MOD_OK
- Purpose: Application document storage and verification

ApplicationRoadmapStep (Many-to-1 with Application)
- Fields: step_order, step_name, instructions, is_complete
- Purpose: Visual progress tracker


D. KARMA SYSTEM
----------------
KarmaTransaction (Immutable Audit Log)
- Fields: student, points, transaction_type, description, related_scholarship, created_at
- Transaction Types:
  * FINDER_BOUNTY: +100 points for scholarship submission approval
  * OTR_COMPLETE: +50 points for completing registration
  * REDEEM_REWARD: -X points for reward purchase
  * PENALTY_FAKE: -100 points for fake submission
  * ADMIN_ADJUST: Manual adjustment
- Purpose: Complete karma history ledger

ScholarshipSubmission (Finder Bounty Program)
- Fields: submitted_by, scholarship_name, organization, website_url, proof_document
- Status: PENDING → APPROVED/REJECTED
- Rewards: karma_awarded, reviewed_by, reviewed_at
- Purpose: Student-submitted scholarship tracking

KarmaReward (Perk Store)
- Fields: reward_name, karma_cost, reward_type, description, stock_quantity, is_active
- Types: VOUCHER (Zomato, Lenskart), BADGE, PRIORITY_SERVICE
- Purpose: Redemption marketplace

RedemptionHistory
- Fields: student, reward, karma_spent, redemption_code, redeemed_at
- Purpose: Reward redemption tracking


E. MODERATION & AUDIT
----------------------
ModeratorActivityLog
- Fields: moderator, application, action_type, details, timestamp
- Actions: VERIFY_DOC, CHANGE_STATUS, NOTE_ADD, ASSIGN_APP
- Purpose: Audit trail for moderator actions

TaskAssignment (1-to-1 with Application)
- Fields: moderator, application, assigned_date, due_date, is_complete
- Purpose: Moderator workload management

AICheckLog
- Fields: application, check_type, input_data (JSON), result_json (JSON), is_success, timestamp
- Types: ELIGIBILITY, DOCUMENT_OCR
- Purpose: AI feature debugging and audit


3. KEY FEATURES & WORKFLOWS
================================================================================

A. ONE-TIME REGISTRATION (OTR) - 7-STEP PROCESS
------------------------------------------------
Step 1: Welcome & Introduction
- Student account creation
- Platform overview

Step 2: Basic Information
- Personal details (name, DOB, gender)
- Contact information

Step 3: Family & Financial Details
- Father/mother names
- Annual income
- Caste category
- Disability status

Step 4: Academic Records
- Degree level, stream, institution
- Current year of study
- Exam scores/CGPA

Step 5: Document Upload (14 Required Documents)
- Aadhaar card
- Income certificate
- Caste certificate (if applicable)
- Disability certificate (if applicable)
- 10th marksheet
- 12th marksheet
- Current semester/year marksheet
- Bank passbook
- Passport photo
- Other supporting documents

Step 6: Banking Information
- Bank account number
- IFSC code
- Bank name
- For scholarship disbursements

Step 7: Review & Submission
- Review all entered information
- Certify accuracy
- Complete OTR → Award +50 karma points

Benefits:
- Complete once, use for all applications
- Auto-fill application forms
- Faster application processing


B. AI-POWERED SCHOLARSHIP MATCHING
-----------------------------------
Algorithm:
1. Parse student profile data
2. Compare against EligibilityCriteria for each scholarship
3. Apply comparison operators (GT, LT, EQ, IN)
4. Calculate match score (0-100%)
5. Rank scholarships by match probability
6. Filter by deadlines and active status

Criteria Examples:
- Income-based: annual_income < 250000
- Academic: last_exam_score >= 70
- Category-based: caste_category IN ['SC', 'ST', 'OBC']
- Age restrictions: age < 25
- Disability benefits: is_disabled == True

Output:
- Smart Discovery Engine shows 95%+ match scholarships first
- Real-time deadline alerts
- Verified funding sources only


C. APPLICATION WORKFLOW
---------------------------------
Student Side:
1. Browse matched scholarships
2. Select scholarship → Create draft application
3. Upload required documents
4. Submit for moderator review
5. Track progress via roadmap steps

Moderator Side:
1. Review application queue
2. Verify uploaded documents
3. AI verification: Automated OCR and validation
4. Manual verification if AI flags issues
5. Approve/reject application
6. Submit to funder organization

Organization (Funder) Side:
1. View submitted applications
2. Review student profiles and documents
3. Make funding decision: Approve/Reject
4. Scholarship disbursement

Earning Karma:

Penalties:
- Fake submission: -100 points
- Incomplete applications: -10 points
- Spam reporting: -50 points

Karma Range: 0-∞ (unlimited ceiling)
Leaderboard: Global ranking based on total points


E. FINDER BOUNTY PROGRAM (MONETIZATION FEATURE)
------------------------------------------------
Concept:
Students discover niche scholarships not in the database and submit them for 
verification. Upon moderator approval, they earn 100 karma points.

Workflow:
1. Student finds scholarship (website/organization)
2. Submits via Karma Dashboard:
   - Scholarship name
   - Organization
   - Website URL
   - Proof document (screenshot/PDF)
3. Moderator reviews submission
4. If APPROVED:
   - Scholarship added to database
   - Student awarded 100 karma points
   - Student credited as "finder"
5. If REJECTED:
   - -100 karma penalty (prevents spam)
   - Admin notes explaining rejection

Database Tracking:
- Scholarship.found_by_student (FK to StudentProfile)
- Scholarship.finder_karma_awarded (int)

5. TECHNICAL IMPLEMENTATION DETAILS
================================================================================

A. AUTHENTICATION & AUTHORIZATION
----------------------------------
- Custom User model (AbstractUser)
- Role-based access control via boolean flags
- Decorators: @login_required, @user_passes_test(is_student)
- Session-based authentication
- LOGIN_REDIRECT_URL: /accounts/dashboard/
- Smart dashboard routing based on user role


B. FILE UPLOAD SYSTEM
----------------------
- MEDIA_ROOT: project_root/media/
- Document paths:
  * Student docs: student_documents/YYYY/MM/
  * Application docs: application_docs/
  * Scholarship submissions: scholarship_submissions/YYYY/MM/
- File size limit: 10MB per file
- Allowed formats: PDF, JPG, PNG, DOCX


C. KARMA TRANSACTION SYSTEM
----------------------------
Utility Functions (karma/utils.py):
- award_karma(): Atomic transaction to add points
- deduct_karma(): Atomic transaction to subtract points
- update_karma_ranks(): Periodic rank calculation (cron job)
- generate_redemption_code(): 16-char unique codes (XXXX-XXXX-XXXX-XXXX)

Concurrency Safety:
- Django ORM F() expressions for atomic updates
- Database transactions for multi-step operations
- Prevents race conditions


D. AI MATCHING ENGINE (PLANNED)
--------------------------------
Inputs:
- Student profile JSON snapshot
- Scholarship eligibility criteria

Process:
1. Extract student attributes (income, GPA, category, etc.)
2. Evaluate each criterion using comparison operators
3. Calculate weighted match score
4. Log to AICheckLog for debugging

Output:
- Match percentage (0-100%)
- Matched/failed criteria breakdown
- Recommended action


E. DOCUMENT OCR (PLANNED)
--------------------------
Technology: Tesseract OCR / Google Cloud Vision API
Purpose:
- Extract text from uploaded documents
- Verify authenticity (income certificates, marksheets)
- Auto-populate form fields
- Flag suspicious documents for moderator review
